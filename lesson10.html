10.1 AI Agents คืออะไร?

AI Agent คือระบบ AI ที่สามารถ รับรู้สภาพแวดล้อม ตัดสินใจ และลงมือกระทำ เพื่อบรรลุเป้าหมายได้อย่างอิสระ แตกต่างจาก AI Model ทั่วไปที่แค่รับ Input และให้ Output โดย Agent จะวางแผน เลือกใช้เครื่องมือ และวนซ้ำจนกว่างานจะสำเร็จ

AI Model (แบบเดิม)

  • รับ Input → ให้ Output ครั้งเดียว
  • ไม่มีความจำระหว่าง Session
  • ไม่สามารถใช้เครื่องมือภายนอก
  • ไม่วางแผนหรือแก้ปัญหาหลายขั้นตอน
  • ตัวอย่าง: ChatGPT แบบ Single Turn

AI Agent (ใหม่)

  • วนซ้ำหลายรอบจนบรรลุเป้าหมาย
  • มี Memory ทั้งระยะสั้นและระยะยาว
  • ใช้ Tools ภายนอก (Search, Code, API)
  • วางแผนและแบ่งงานเป็นขั้นตอนย่อย
  • ตัวอย่าง: GitHub Copilot Agent, Devin

สูตรของ AI Agent

AI Agent = LLM + Tools + Memory + Planning + Action
การผสมผสานของโมเดลภาษา เครื่องมือภายนอก ความจำ ความสามารถในการวางแผน และการลงมือกระทำ คือสิ่งที่ทำให้ Agent ทรงพลังกว่า AI Model ธรรมดา

Agent Loop: วงจรการทำงานของ Agent

Perceive

รับรู้ Input จากผู้ใช้ สภาพแวดล้อม หรือผลลัพธ์ก่อนหน้า

Think (LLM)

ประมวลผลด้วย LLM วิเคราะห์ วางแผน และตัดสินใจว่าจะทำอะไรต่อ

Act (Tools)

เรียกใช้ Tools เช่น ค้นหาเว็บ รันโค้ด เรียก API หรืออ่านไฟล์

Observe

รับผลลัพธ์จาก Tools แล้วนำกลับไปเป็น Input ในรอบถัดไป

วนซ้ำ

ประเภทของ AI Agents

Reactive Agent

ตอบสนองต่อ Input โดยตรง ไม่มีการวางแผนล่วงหน้า เร็วแต่ขาดความยืดหยุ่น เหมาะกับงาน Rule-based

Deliberative Agent

วางแผนก่อนลงมือ ใช้ Model ของโลกภายใน คำนวณลำดับขั้นตอนเพื่อบรรลุเป้าหมาย

Hybrid Agent

ผสมผสาน Reactive และ Deliberative ตอบสนองเร็วสำหรับงานง่าย วางแผนสำหรับงานซับซ้อน

Multi-Agent System

หลาย Agent ทำงานร่วมกัน แต่ละตัวมีความเชี่ยวชาญต่างกัน ประสานงานเพื่อแก้ปัญหาที่ซับซ้อน

GitHub Copilot Agent

Agent ที่ช่วยเขียนโค้ด ค้นหา Bug แนะนำ Refactoring และสร้าง PR อัตโนมัติใน Repository

10.2 LangChain Agents

LangChain คือ Framework โอเพนซอร์สสำหรับสร้าง Application ที่ใช้ LLM เป็นแกนหลัก มีโครงสร้างหลักประกอบด้วย Chains, Agents, Tools และ Memory ทำให้การสร้าง AI Application ซับซ้อนเป็นเรื่องง่าย

สถาปัตยกรรม LangChain Agent

Agent

ตัวควบคุมหลัก ตัดสินใจว่าจะใช้ Tool ใด และเมื่อใดจะหยุด

LLM

โมเดลภาษา (GPT-4o, Claude ฯลฯ) ที่ใช้ reason และตัดสินใจ

Tools

Search, Calculator, Python REPL, File System, API Calls

Memory

บันทึกบทสนทนาและผลลัพธ์เพื่อใช้ใน Context ถัดไป

ตัวอย่างโค้ด: LangChain ReAct Agent

Python — LangChain Agent with Custom Tools Python 
from langchain_openai import ChatOpenAI
from langchain.agents import create_react_agent, AgentExecutor
from langchain.tools import tool
from langchain import hub

# Define custom tools
@tool
def calculate(expression: str) -> str:
    """Calculate a mathematical expression. Input: math expression as string"""
    try:
        result = eval(expression)  # Note: use safer eval in production
        return str(result)
    except Exception as e:
        return f"Error: {e}"

@tool
def search_web(query: str) -> str:
    """Search for information on the web"""
    # Mock implementation - use actual search API
    return f"Search results for: {query}"

# Create agent
llm = ChatOpenAI(model="gpt-4o", temperature=0)
tools = [calculate, search_web]
prompt = hub.pull("hwchase17/react")

agent = create_react_agent(llm, tools, prompt)
agent_executor = AgentExecutor(
    agent=agent,
    tools=tools,
    verbose=True,
    max_iterations=5
)

result = agent_executor.invoke({
    "input": "คำนวณ 15% ของ 2500 บาท แล้วบอกว่าเพียงพอสำหรับอาหารกี่มื้อ"
})
print(result["output"])
ReAct Pattern = Reasoning + Acting
ReAct คือ Pattern การทำงานของ Agent ที่สลับระหว่าง การคิด (Thought) และ การกระทำ (Action) Agent จะอธิบายเหตุผลก่อน จากนั้นเลือก Tool ที่เหมาะสม ดูผลลัพธ์ แล้วตัดสินใจในรอบถัดไปจนกว่างานจะสำเร็จ ทำให้ง่ายต่อการ Debug และเข้าใจการทำงานของ Agent

Components หลักของ LangChain

Chains

การเชื่อมต่อ Components หลายตัวเข้าด้วยกัน เช่น Prompt → LLM → Parser หรือการสร้าง Pipeline ที่ซับซ้อน ใช้สำหรับ Workflow ที่กำหนดขั้นตอนล่วงหน้า

Agents

ใช้ LLM ในการตัดสินใจว่าจะทำอะไรต่อ เลือก Tool เหมาะสม ทำงานแบบ Dynamic ต่างจาก Chain ที่กำหนดขั้นตอนตายตัว

Tools

ฟังก์ชันที่ Agent เรียกใช้ได้ เช่น ค้นหาเว็บ คำนวณ รันโค้ด Python อ่านไฟล์ เรียก API ภายนอก ฯลฯ ต้องมี Docstring อธิบายวิธีใช้

Memory

ระบบความจำของ Agent มีทั้ง Buffer Memory (จำบทสนทนาล่าสุด) Summary Memory (สรุปประวัติ) และ Vector Store Memory (ค้นหาจาก Long-term memory)

10.3 Multi-Agent Systems

Multi-Agent System (MAS) คือระบบที่มี AI Agent หลายตัวทำงานร่วมกัน แต่ละตัวมีความเชี่ยวชาญเฉพาะด้าน ช่วยแก้ปัญหาที่ซับซ้อนเกินกว่า Agent เดี่ยวจะรับมือได้ด้วยการแบ่งงานและประสานงาน

รูปแบบ Multi-Agent Systems

Hierarchical (Manager + Workers)

มี Manager Agent ที่รับงานจากผู้ใช้ แบ่งงานให้ Worker Agents ที่เชี่ยวชาญแต่ละด้าน แล้วรวมผลลัพธ์ เหมาะกับงาน Complex ที่แบ่งได้ชัดเจน เช่น Research → Write → Edit

Collaborative

Agent หลายตัวทำงานร่วมกันในฐานะเพื่อนร่วมทีม แลกเปลี่ยนข้อมูล ช่วยตรวจสอบงานกัน ไม่มีลำดับชั้น เหมาะสำหรับงานที่ต้องการมุมมองหลากหลาย

Competitive

Agent แข่งขันกันเสนอคำตอบ ระบบเลือกคำตอบที่ดีที่สุด ใช้หลัก Diversity เพื่อหลีกเลี่ยง Bias จาก Agent เดี่ยว เช่น ระบบ Debate ระหว่าง Agent

ตัวอย่างโค้ด: Multi-Agent ด้วย CrewAI

Python — CrewAI Multi-Agent System Python 
from crewai import Agent, Task, Crew

# Define specialized agents
researcher = Agent(
    role="Research Analyst",
    goal="Research and gather accurate information",
    backstory="Expert researcher with access to internet",
    llm=ChatOpenAI(model="gpt-4o"),
    verbose=True
)

writer = Agent(
    role="Content Writer",
    goal="Write clear and engaging content in Thai",
    backstory="Professional Thai content writer",
    llm=ChatOpenAI(model="gpt-4o"),
    verbose=True
)

editor = Agent(
    role="Editor",
    goal="Review and improve content quality",
    backstory="Experienced editor ensuring accuracy",
    llm=ChatOpenAI(model="gpt-4o"),
    verbose=True
)

# Define tasks
research_task = Task(
    description="Research the latest trends in Generative AI for 2025",
    agent=researcher
)

write_task = Task(
    description="Write a 500-word Thai article based on research",
    agent=writer
)

edit_task = Task(
    description="Review and polish the article",
    agent=editor
)

# Create crew and run
crew = Crew(
    agents=[researcher, writer, editor],
    tasks=[research_task, write_task, edit_task],
    verbose=True
)

result = crew.kickoff()
print(result)

เมื่อไหร่ควรใช้ Multi-Agent?
ใช้ Multi-Agent เมื่องานซับซ้อนจนยากสำหรับ Agent เดียว งานต้องการความเชี่ยวชาญหลายด้านพร้อมกัน ต้องการระบบตรวจสอบและ Review ในตัว หรือต้องการให้ Agent ทำงานขนานกันเพื่อประหยัดเวลา

10.4 n8n: Visual AI Automation

n8n (pronounced "n-eight-n") คือ Low-code Workflow Automation Platform แบบ Open Source ที่สามารถ Self-host ได้ ช่วยให้สร้าง Workflow อัตโนมัติโดยเชื่อมต่อ Apps และ APIs หลายร้อยตัวผ่าน Visual Interface แบบ Node-based โดยไม่ต้องเขียนโค้ดมาก

n8n คืออะไร?
n8n เป็น Fair-code Licensed Automation Tool ที่รัน Self-hosted บน Server ของคุณได้ ไม่มีค่า Subscription รองรับ Integrations กว่า 350+ Apps มี AI Nodes ในตัว (OpenAI, Anthropic, Google AI) และสามารถรัน JavaScript หรือ Python Code ได้ภายใน Workflow

10.5 Make & Zapier with AI

Make (Integromat)

  • Visual Scenario Builder แบบ Canvas
  • ราคาถูกกว่า Zapier
  • รองรับ Data Transformation ซับซ้อน

  • AI Integrations: OpenAI, Claude, Google AI

Zapier

  • ใช้งานง่ายที่สุด สร้าง Zap ใน 5 นาที
  • Integrations มากที่สุด (5000+ Apps)
  • AI-powered Zap Builder
  • มี Zapier AI Actions ในตัว
  • เหมาะสำหรับ Business User ไม่ต้องโค้ด
  • ราคาสูงกว่าแต่ง่ายที่สุด

10.6 AutoGPT & Autonomous Agents

AutoGPT คือ Autonomous AI Agent ตัวแรกที่ได้รับความสนใจจากสาธารณชนในวงกว้าง เปิดตัวในปี 2023 โดย Toran Bruce Richards ใช้แนวคิดว่า แค่บอกเป้าหมายระยะยาวให้ AI แล้วให้มันวางแผนและทำงานเองทั้งหมด นับเป็นจุดเปลี่ยนสำคัญของการพัฒนา Autonomous AI

10.7 Best Practices & Design Patterns

การออกแบบ AI Agent ที่ดีต้องเลือก Pattern ที่เหมาะสมกับปัญหา มีระบบ Monitoring บริหารต้นทุน และคำนึงถึงความปลอดภัยเสมอ

Agent Design Patterns ที่สำคัญ

1. ReAct (Reason + Act)

Agent สลับระหว่าง Thought (การคิด/วิเคราะห์) และ Action (การกระทำ/ใช้ Tool) ทำให้เห็น Reasoning Chain ชัดเจน ง่ายต่อ Debug เหมาะกับงานที่ต้องการความโปร่งใสสูง เช่น การตอบคำถามที่ต้องค้นหาข้อมูล

2. Plan-and-Execute

Agent วางแผนทุกขั้นตอนก่อน แล้วค่อยดำเนินการตามแผน เหมาะสำหรับงานที่ซับซ้อนและมีหลายขั้นตอนชัดเจน ลด Hallucination เพราะมีโครงสร้างชัดเจน แต่อาจยืดหยุ่นน้อยกว่า ReAct

3. Reflection

Agent ตรวจสอบและวิจารณ์ Output ของตัวเองก่อนส่งให้ผู้ใช้ อาจมี Critic Agent แยกต่างหาก ช่วยเพิ่มคุณภาพผลลัพธ์ ลด Error แต่เพิ่ม Latency และ Cost เหมาะกับงานที่ต้องการความแม่นยำสูง

4. Tool-Use

Agent เลือกใช้ Tool ที่เหมาะสมจากคลัง (Search, Calculator, API, Code) ควรออกแบบ Tool ให้มี Docstring ชัดเจน ชื่อ Tool สื่อความหมาย และจำกัดจำนวน Tool ต่อ Agent ไม่ให้มากเกินไป

5. Memory

ผสมผสาน Short-term Memory (Context Window) กับ Long-term Memory (Vector DB เช่น Pinecone, Weaviate) ช่วยให้ Agent จำข้อมูลสำคัญข้ามบทสนทนา เหมาะสำหรับ Personalized Assistant หรืองานระยะยาว

Cost Management สำหรับ AI Agents

เทคนิคควบคุมต้นทุน
  • กำหนด max_iterations เสมอ ป้องกัน Agent วนซ้ำไม่สิ้นสุด
  • ใช้โมเดลเล็ก (GPT-4o-mini, Claude Haiku) สำหรับ Task ง่าย ใช้โมเดลใหญ่เฉพาะ Task ยาก
  • Cache ผลลัพธ์ที่ใช้บ่อยด้วย Semantic Cache เพื่อไม่ต้องเรียก API ซ้ำ
  • ใช้ Streaming เพื่อแสดงผลระหว่างรอ ไม่ใช่เพิ่มความเร็ว
  • ตั้ง Budget Alert บน OpenAI/Anthropic Dashboard

Safety Considerations

หลักความปลอดภัยสำหรับ AI Agents
  • Human-in-the-Loop — ให้มนุษย์อนุมัติก่อน Action ที่มีผลกระทบสูง เช่น ส่ง Email, ลบไฟล์, ทำธุรกรรม
  • Sandboxing — รัน Code ใน Isolated Environment ป้องกัน Agent เข้าถึง System จริง
  • Permission Scoping — ให้ Agent มีสิทธิ์น้อยที่สุดที่จำเป็น (Principle of Least Privilege)
  • Audit Logging — บันทึกทุก Action ที่ Agent ทำเพื่อตรวจสอบย้อนหลัง
  • Rate Limiting — จำกัดความถี่ของ API Calls เพื่อป้องกัน Accidental DDoS

10.8 สรุปและแบบฝึกหัด

สรุปสาระสำคัญ

  • AI Agent = LLM + Tools + Memory + Planning + Action ทำงานวนซ้ำเพื่อบรรลุเป้าหมาย ต่างจาก AI Model ที่ตอบครั้งเดียว
  • LangChain เป็น Framework ยอดนิยมสำหรับสร้าง Agent ใช้ ReAct Pattern ที่สลับ Thought และ Action
  • Multi-Agent Systems เช่น CrewAI ช่วยแก้ปัญหาซับซ้อนด้วยการแบ่ง Agent ให้มีบทบาทเฉพาะ
  • n8n, Make, Zapier คือ No/Low-code Automation Tools ที่รองรับ AI Integration ใช้สร้าง Workflow โดยไม่ต้องเขียนโค้ดมาก
  • Autonomous Agents เช่น AutoGPT สามารถทำงานเองจนสำเร็จ แต่ต้องระวังเรื่องต้นทุน ความปลอดภัย และ Hallucination
  • ควรใช้ Human-in-the-Loop, Sandboxing และ Monitoring เสมอเมื่อ Deploy Agent ใน Production
  • Design Patterns สำคัญ ได้แก่ ReAct, Plan-and-Execute, Reflection, Tool-Use และ Memory

แบบฝึกหัดท้ายบท

1
สร้าง LangChain Agent พร้อม Tools

สร้าง Agent ด้วย LangChain ที่มี Custom Tools อย่างน้อย 3 อย่าง เช่น ค้นหาเว็บ คำนวณ และอ่านไฟล์ CSV ทดสอบโดยถามคำถามที่ต้องใช้ Tools หลายตัวร่วมกัน บันทึก Log และวิเคราะห์ ReAct Trace

2
สร้าง Multi-Agent System ด้วย CrewAI

สร้างระบบ 2 Agent (Researcher + Writer) ด้วย CrewAI เพื่อค้นหาข้อมูลและเขียนบทความภาษาไทย กำหนด Role, Goal และ Backstory ให้ชัดเจน เปรียบเทียบผลลัพธ์กับการใช้ Agent เดี่ยว

3
สร้าง Workflow ใน n8n สำหรับสรุป Email

ติดตั้ง n8n บน Local หรือใช้ Cloud Version สร้าง Workflow ที่รับ Email ใหม่ ส่งไปให้ OpenAI สรุปเนื้อหา แล้วส่งสรุปไปยัง Slack Channel ทดสอบกับ Email จริงอย่างน้อย 5 ฉบับ

4
สร้าง Simple Autonomous Agent สำหรับโจทย์คณิตศาสตร์

สร้าง Agent ที่รับโจทย์คณิตศาสตร์แบบ Word Problem รับเป้าหมายและวางแผนขั้นตอนการแก้ปัญหา ใช้ Calculator Tool คำนวณ และตรวจสอบคำตอบด้วย Reflection Pattern กำหนด max_iterations = 5

5
Implement Memory ด้วย Vector Database

เพิ่ม Long-term Memory ให้กับ Agent โดยใช้ Vector Database (Chroma หรือ FAISS) ให้ Agent จำข้อมูลผู้ใช้ข้าม Session ได้ ทดสอบโดยถามข้อมูลที่เคยบอกในบทสนทนาก่อนหน้า วัดประสิทธิภาพการค้นหาด้วย Cosine Similarity Score
