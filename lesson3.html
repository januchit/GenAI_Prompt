3.1 Prompt Engineering คืออะไร?

Prompt Engineering คือศาสตร์และศิลป์ของการออกแบบคำสั่ง (Prompt) เพื่อสื่อสารกับ AI อย่างมีประสิทธิภาพ เพื่อให้ได้ผลลัพธ์ที่ถูกต้อง ครบถ้วน และตรงความต้องการ

เปรียบเหมือนการสั่งงานคน

"ช่วยทำรายงาน"
→ ไม่ชัดเจน ได้งานผิดที่คาดหวัง

"ช่วยทำรายงานสรุปยอดขายเดือนมกราคม รูปแบบตาราง ภาษาไทย ความยาว 1 หน้า A4"
→ ชัดเจน ได้งานตรงใจ

เปรียบเหมือนการตั้งโปรแกรม

Prompt คือ "Input" ที่กำหนดพฤติกรรมของ AI ในการสนทนานั้น Prompt ที่ดี = Output ที่แม่นยำ

Prompt ที่แย่จะทำให้ AI เดา ตอบกว้าง หรือตีความผิด

ทำไม Prompt Engineering ถึงสำคัญ?

ความแม่นยำ

Prompt ที่ดีลดความเข้าใจผิดและเพิ่มความถูกต้องของคำตอบ

ประหยัดเวลา

ไม่ต้องถามซ้ำ ได้คำตอบที่ใช้งานได้จริงตั้งแต่ครั้งแรก

ปลดล็อกความสามารถ

AI มีความสามารถซ่อนอยู่มาก Prompt ที่ดีช่วยดึงออกมาใช้ได้

ประหยัดต้นทุน

Prompt สั้นกระชับลด Token ที่ใช้ ลดค่าใช้จ่าย API

3.2 โครงสร้างของ Prompt

Prompt ที่ดีมักประกอบด้วยองค์ประกอบเหล่านี้ (ไม่จำเป็นต้องครบทุกข้อ):

Role

บทบาท (Role / Persona)

กำหนดให้ AI เล่นบทบาทหรือมีความเชี่ยวชาญในด้านใด

"คุณเป็นอาจารย์สอนวิชาคณิตศาสตร์ระดับมหาวิทยาลัย..."
Context

บริบท (Context / Background)

ให้ข้อมูลพื้นฐานที่ AI จำเป็นต้องรู้เพื่อตอบได้ถูกต้อง

"ฉันกำลังเตรียม presentation สำหรับผู้บริหารระดับ C-Suite ที่ไม่มีพื้นฐาน AI..."
Task

งาน (Task / Instruction)

บอกชัดเจนว่าต้องการให้ AI ทำอะไร — หัวใจสำคัญของ Prompt

"สรุปประโยชน์ของ Generative AI ในองค์กร 5 ข้อ พร้อมตัวอย่างจริง"
Format

รูปแบบผลลัพธ์ (Output Format)

กำหนดว่าต้องการคำตอบในรูปแบบใด เช่น ตาราง, bullet, JSON, โค้ด

"ตอบเป็น Markdown ใช้หัวข้อ H2 สำหรับแต่ละประเด็น ความยาวไม่เกิน 500 คำ"
Example

ตัวอย่าง (Examples)

ให้ตัวอย่าง Input/Output เพื่อช่วยให้ AI เข้าใจรูปแบบที่ต้องการ

"ตัวอย่าง: Input: 'แมว' → Output: 'สัตว์เลี้ยงลูกด้วยนม 4 ขา...'"
Constraint

ข้อจำกัด (Constraints)

บอกสิ่งที่ไม่ต้องการหรือข้อห้ามในการตอบ

"อย่าใช้ศัพท์เทคนิค อย่าแนะนำให้ซื้อสินค้า ตอบเป็นภาษาไทยเท่านั้น"

ตัวอย่าง Prompt แบบครบโครงสร้าง

Full Prompt Example
[Role] คุณเป็นนักการตลาดดิจิทัลที่มีประสบการณ์ 10 ปี

[Context] ฉันมีร้านขายกาแฟขนาดเล็กในกรุงเทพฯ มีลูกค้าหลักอายุ 25-35 ปี งบโฆษณา 5,000 บาท/เดือน

[Task] ช่วยวางแผนการตลาดบน Social Media สำหรับเดือนหน้า

[Format] ตอบเป็นตาราง แบ่งตาม Platform (IG, TikTok, Facebook) พร้อมระบุ: ประเภทคอนเทนต์, ความถี่โพสต์, งบที่แนะนำ

[Constraint] ใช้ภาษาไทย ไม่ต้องอธิบายทฤษฎี เน้นแผนที่ทำได้จริงทันที

3.3 หลัก 6 ข้อของ Prompt ที่ดี

1

ชัดเจน (Clear)

ใช้ภาษาตรงไปตรงมา ไม่คลุมเครือ ระบุสิ่งที่ต้องการอย่างชัดเจน

"บอกเกี่ยวกับ AI หน่อย"
"อธิบายว่า Generative AI แตกต่างจาก Traditional AI อย่างไร ในแง่เทคนิคและการใช้งาน ความยาว 3-4 ประโยค"
2

มีบริบท (Contextual)

ให้ข้อมูลพื้นหลังที่จำเป็น เพื่อให้ AI ตอบได้เหมาะกับสถานการณ์จริง

"แปลประโยคนี้"
"แปลประโยคนี้จากอังกฤษเป็นไทย สำหรับใช้ใน Email ทางธุรกิจ ให้เป็นทางการและสุภาพ: 'We need to reschedule the meeting.'"
3

ระบุขอบเขต (Specific)

กำหนดความยาว รูปแบบ จำนวน หรือขอบเขตของคำตอบที่ต้องการ

"ช่วยเขียนบทความ"
"เขียนบทความ 300 คำ หัวข้อ 'AI กับการศึกษาไทย' สำหรับนิตยสาร กลุ่มเป้าหมายคือครูและอาจารย์"
4

กำหนด Tone (Tone & Style)

บอกน้ำเสียง สไตล์ และระดับภาษาที่ต้องการ เพื่อให้ตรงกลุ่มเป้าหมาย

🎓 เป็นทางการ😊 เป็นกันเอง😂 ขำขัน 🔬 วิชาการ📣 กระตุ้น🤝 สร้างแรงบันดาลใจ
5

Iterate (ปรับปรุงต่อเนื่อง)

Prompt Engineering เป็นกระบวนการ ทดลอง → ดูผล → ปรับ → ทดลองใหม่

เขียน Prompt ดูผลลัพธ์ วิเคราะห์ปัญหา ปรับ Prompt
6

ระบุข้อห้าม (Negative Constraints)

บอก AI ว่า "ไม่ต้องการอะไร" ช่วยลด Hallucination และคำตอบที่ไม่เกี่ยวข้อง

"ตอบเฉพาะข้อมูลที่แน่ใจ ถ้าไม่มั่นใจให้บอกว่าไม่ทราบ อย่าสมมติข้อมูล"

3.4 Role Prompting

การกำหนด Role ให้ AI ช่วยปรับโทน ความเชี่ยวชาญ และรูปแบบการตอบให้เหมาะกับงาน เป็นเทคนิคที่ง่ายแต่ได้ผลชัดเจน

System Prompt vs User Prompt

System Prompt

กำหนดบุคลิก กฎ และพฤติกรรมหลักของ AI ตลอดการสนทนา ผู้ใช้ทั่วไปมักมองไม่เห็น

"คุณเป็น Customer Service ของร้านอาหาร 'ข้าวต้มป้าแดง' ตอบภาษาไทยเท่านั้น สุภาพและเป็นมิตร ถ้าถามนอกเรื่องอาหารให้ปฏิเสธสุภาพ"
User Prompt

คำถามหรือคำสั่งที่ผู้ใช้พิมพ์ในแต่ละรอบ AI จะตอบโดยอ้างอิงทั้ง System และ User Prompt

"มีเมนูอะไรบ้างที่ไม่มีผักชี?"

ตัวอย่าง Role Prompting ในบริบทต่างๆ

แพทย์

"คุณเป็นแพทย์ผู้เชี่ยวชาญด้านโภชนาการ อธิบายด้วยภาษาที่เข้าใจง่าย..."

ทนายความ

"คุณเป็นทนายความผู้เชี่ยวชาญกฎหมายแรงงานไทย อธิบายสิทธิ์ของลูกจ้าง..."

ครูสอนพิเศษ

"คุณเป็นครูสอนคณิตศาสตร์ ม.4 อธิบายให้เข้าใจง่าย พร้อมตัวอย่าง 3 ข้อ..."

นักวิเคราะห์

"คุณเป็น Business Analyst วิเคราะห์ข้อมูลต่อไปนี้และสรุปเป็น Insight..."

3.5 Output Format Control

การควบคุมรูปแบบผลลัพธ์ช่วยให้นำ Output ไปใช้งานต่อได้ทันที ไม่ต้องแก้ไขเพิ่มเติม

ตาราง (Table)
Prompt: "เปรียบเทียบ ChatGPT, Claude, Gemini ในรูปแบบตาราง Markdown"
| โมเดล   | ผู้พัฒนา  | จุดเด่น          |
|---------|----------|-----------------|
| ChatGPT | OpenAI   | ใช้งานง่าย       |
| Claude  | Anthropic| Context ยาว     |
| Gemini  | Google   | ค้นข้อมูลได้    |
JSON
Prompt: "สร้างข้อมูลนักเรียน 2 คน ในรูปแบบ JSON"
{
  "students": [
    {"id": 1, "name": "สมชาย", "gpa": 3.8},
    {"id": 2, "name": "สมหญิง", "gpa": 3.5}
  ]
}
Numbered List
Prompt: "แนะนำขั้นตอนการเรียน Python 5 ขั้นตอน เป็น numbered list"
  1. เรียนพื้นฐาน Python syntax
  2. ฝึก Data Types และ Control Flow
  3. เรียน Functions และ OOP
  4. ทำ Projects จริง
  5. เรียน Library เช่น NumPy, Pandas
Markdown
Prompt: "เขียนรายงานสรุป AI ในรูปแบบ Markdown มีหัวข้อ บทนำ เนื้อหา สรุป"
# รายงาน: AI ในปัจจุบัน
## บทนำ
AI กำลังเปลี่ยนแปลงโลก...
## เนื้อหา
### 1. LLM
...
## สรุป
AI จะมีบทบาทสำคัญ...
เทคนิค: บอกรูปแบบด้วยตัวอย่าง
แทนที่จะอธิบายรูปแบบยาวๆ ให้ใส่ตัวอย่างรูปแบบที่ต้องการไปเลย เช่น "ตอบในรูปแบบนี้: [หัวข้อ]: [คำอธิบาย]"

3.6 ข้อผิดพลาดที่ควรหลีกเลี่ยง

Prompt คลุมเครือเกินไป

การใช้คำถามกว้างๆ ทำให้ AI ตอบได้ทุกทิศทาง ผลลัพธ์ไม่ตรงใจ

"บอกเกี่ยวกับ Machine Learning"
"อธิบาย Supervised Learning คืออะไร ด้วยตัวอย่าง 2 ตัวอย่าง ความยาว 100 คำ"

ขอหลายอย่างในคราวเดียว

Prompt ที่มีหลาย Task ซ้อนกัน ทำให้ AI ทำได้ไม่ครบหรือไม่ดีในทุกส่วน

"เขียนบทความ แปลเป็นอังกฤษ สรุป และทำ Presentation"
แบ่งเป็น Prompt แยกกัน ทีละ Task

ไม่ระบุกลุ่มเป้าหมาย

AI ไม่รู้ว่าควรใช้ระดับภาษาใด เทคนิคหรือทั่วไป เด็กหรือผู้ใหญ่

"อธิบาย Neural Network"
"อธิบาย Neural Network ให้นักเรียน ม.5 เข้าใจ โดยไม่ใช้สมการ"

เชื่อทุกอย่างที่ AI บอก

LLM สามารถ "Hallucinate" — สร้างข้อมูลที่ดูน่าเชื่อถือแต่ผิด ควรตรวจสอบข้อเท็จจริงเสมอ

ใช้ Prompt: "ตอบเฉพาะที่แน่ใจ ถ้าไม่มั่นใจให้บอก" เพื่อลด Hallucination

ไม่ Iterate

ถ้าผลลัพธ์แรกไม่ดี หลายคนยอมแพ้ แต่ควรปรับ Prompt และลองใหม่ การ Iterate คือกุญแจสำคัญ

3.7 สรุปและแบบฝึกหัด

สรุปสาระสำคัญ

  • Prompt Engineering คือการออกแบบคำสั่งเพื่อดึงประสิทธิภาพสูงสุดจาก AI
  • โครงสร้าง Prompt ที่ดี: Role + Context + Task + Format + Example + Constraint
  • หลัก 6 ข้อ: ชัดเจน, มีบริบท, ระบุขอบเขต, กำหนด Tone, Iterate, ระบุข้อห้าม
  • Role Prompting ช่วยให้ AI ตอบตรงจุดและมีความเชี่ยวชาญในด้านที่ต้องการ
  • ควบคุม Output Format เพื่อได้ผลลัพธ์ที่นำไปใช้ได้ทันที
  • หลีกเลี่ยง Prompt คลุมเครือ, หลาย Task พร้อมกัน, และเชื่อ AI โดยไม่ตรวจสอบ

แบบฝึกหัดท้ายบท

1
ปรับปรุง Prompt

นำ Prompt แย่ๆ เหล่านี้ไปปรับปรุงให้ดีขึ้น: (1) "แนะนำหนังสือ" (2) "เขียน Email" (3) "อธิบาย AI" แล้วทดสอบกับ ChatGPT หรือ Claude

2
Role Prompting ทดลอง

ถามคำถามเดียวกัน "อธิบาย Blockchain" กับ AI โดยกำหนด Role ต่างกัน: (1) ครูสอนเด็ก ป.6 (2) นักลงทุน (3) นักพัฒนาซอฟต์แวร์ เปรียบเทียบความแตกต่าง

3
สร้าง System Prompt

ออกแบบ System Prompt สำหรับ AI Chatbot ของธุรกิจที่ตนเองสนใจ (เช่น ร้านค้า, คลินิก, โรงเรียน) กำหนดบุคลิก กฎ และขอบเขตการตอบ

4
Output Format Challenge

ใช้ Prompt เดียวให้ AI สร้างข้อมูลในทุกรูปแบบนี้: ตาราง, JSON, Markdown, Bullet points เปรียบเทียบว่ารูปแบบไหนเหมาะกับการใช้งานอะไร
