4.1 Zero-shot Prompting

Zero-shot คือการถาม AI โดยไม่ให้ตัวอย่างใดๆ เลย เพียงแค่บอกว่าต้องการอะไร ขึ้นอยู่กับความรู้ที่โมเดลเรียนรู้มาจาก Pre-training

Zero-shot
ตัวอย่าง Zero-shot Prompt
จงจำแนกประเภทของข้อความต่อไปนี้ว่าเป็น Positive, Negative หรือ Neutral:

"สินค้าดีมาก ส่งเร็ว บริการประทับใจ ซื้ออีกแน่นอน"
AI Response
Positive
เหตุผล: ข้อความมีคำแสดงความพึงพอใจหลายจุด เช่น "ดีมาก", "ประทับใจ", "ซื้ออีกแน่นอน"

เมื่อไรควรใช้ Zero-shot?

เหมาะกับ
  • งานทั่วไปที่ AI รู้อยู่แล้ว
  • สรุป, แปล, จัดหมวดหมู่
  • เมื่อต้องการคำตอบเร็ว
ไม่เหมาะกับ
  • งานเฉพาะทางที่ซับซ้อน
  • ต้องการรูปแบบเฉพาะ
  • Domain ที่ AI ไม่คุ้นเคย

4.2 Few-shot Prompting

Few-shot คือการให้ตัวอย่าง Input→Output จำนวน 2–5 ชุดก่อนถามคำถามจริง ช่วยให้ AI เข้าใจรูปแบบที่ต้องการได้แม่นยำขึ้นมาก

Few-shot
ตัวอย่าง Few-shot Prompt
// ตัวอย่างที่ 1
Input: "บริษัทยกเลิกการประชุมกะทันหัน"
Sentiment: Negative | อารมณ์: หงุดหงิด

// ตัวอย่างที่ 2
Input: "ได้รับโบนัสเพิ่มขึ้น 20%"
Sentiment: Positive | อารมณ์: ดีใจมาก

// ตัวอย่างที่ 3
Input: "การประชุมวันนี้ใช้เวลา 2 ชั่วโมง"
Sentiment: Neutral | อารมณ์: เฉยๆ

// คำถามจริง
Input: "เพื่อนร่วมงานช่วยทำงานส่วนที่ยากให้โดยไม่ได้ขอ"
Sentiment: ??? | อารมณ์: ???
AI Response
Sentiment: Positive | อารมณ์: ประทับใจและขอบคุณ

หลักการเลือก Few-shot Examples

  • จำนวน: 2–5 ตัวอย่างก็เพียงพอ มากไปทำให้ Prompt ยาวเกินไป
  • ความหลากหลาย: ครอบคลุมกรณีต่างๆ ที่อาจเกิดขึ้น
  • ความถูกต้อง: ตัวอย่างต้องถูกต้อง เพราะ AI จะเรียนรู้ตามนั้น
  • รูปแบบสม่ำเสมอ: ใช้ Format เดียวกันทุก Example
One-shot vs Few-shot
One-shot = ให้ตัวอย่าง 1 ชุด | Few-shot = ให้ 2-5 ชุด | Many-shot = ให้มากกว่า 5 ชุด
งานวิจัยพบว่า Few-shot มักให้ผลดีกว่า One-shot อย่างชัดเจน

4.3 Chain-of-Thought (CoT) Prompting

Chain-of-Thought (CoT) คือการกระตุ้นให้ AI แสดงขั้นตอนการคิดทีละขั้น ก่อนตอบคำถาม ช่วยเพิ่มความแม่นยำสูงมากในงานที่ต้องใช้ Reasoning เช่น คณิตศาสตร์, Logic, Multi-step Problems

Chain-of-Thought
ไม่ใช้ CoT
โรงงานผลิตสินค้าได้ 250 ชิ้น/วัน ทำงาน 5 วัน/สัปดาห์ ต้องการสินค้า 3,000 ชิ้น ใช้เวลากี่สัปดาห์?
ใช้เวลา 3 สัปดาห์ (ผิด!)
ใช้ CoT
โรงงานผลิตสินค้าได้ 250 ชิ้น/วัน ทำงาน 5 วัน/สัปดาห์ ต้องการสินค้า 3,000 ชิ้น ใช้เวลากี่สัปดาห์?

Let's think step by step.
ขั้นที่ 1: ผลิต/สัปดาห์ = 250 × 5 = 1,250 ชิ้น
ขั้นที่ 2: จำนวนสัปดาห์ = 3,000 ÷ 1,250 = 2.4 สัปดาห์
ขั้นที่ 3: ปัดขึ้น = 3 สัปดาห์ (แต่ไม่ครบ 3,000)
ดังนั้นต้องใช้ 3 สัปดาห์ เพื่อให้ครบ

วิธีเปิดใช้ CoT

Magic Phrase
"Let's think step by step."
ภาษาไทย
"คิดทีละขั้นตอน"
Explicit
"อธิบายขั้นตอนการแก้ปัญหาก่อนตอบ"
Few-shot CoT
ให้ตัวอย่างที่มีการแสดงขั้นตอนด้วย

Zero-shot CoT vs Few-shot CoT

Zero-shot CoT: เพิ่มแค่ "คิดทีละขั้น" ง่าย ไม่ต้องเตรียมตัวอย่าง เหมาะกับงานทั่วไป
Few-shot CoT: ให้ตัวอย่างที่มีขั้นตอนด้วย แม่นยำกว่า เหมาะกับงานเฉพาะทาง

4.4 Tree-of-Thought (ToT)

Tree-of-Thought เป็นการขยาย CoT โดยให้ AI สำรวจ หลายแนวทางการคิด พร้อมกัน เหมือนต้นไม้ที่แตกกิ่งก้าน แล้วประเมินว่าแนวทางไหนดีที่สุด เหมาะกับปัญหาที่ซับซ้อนและมีหลายวิธีแก้

Tree-of-Thought
ปัญหา
แนวทาง A
A1 ✓
A2 ✗
แนวทาง B ⭐
B1 ✓✓
B2 ✓
แนวทาง C
C1 ✗
C2 ✗
เลือก แนวทาง B1 เป็นคำตอบสุดท้าย
ตัวอย่าง ToT Prompt
ลองคิดวิธีแก้ปัญหาต่อไปนี้ 3 แนวทาง แต่ละแนวทางให้อธิบายขั้นตอน แล้วประเมินข้อดีข้อเสีย สุดท้ายเลือกแนวทางที่ดีที่สุด:

ปัญหา: บริษัท Startup ต้องการเพิ่ม User ใน App โดยมีงบ 50,000 บาทต่อเดือน
เมื่อไรควรใช้ ToT?
ปัญหาการวางกลยุทธ์, การออกแบบระบบ, Creative Problem Solving ที่มีหลายวิธี ToT เหมาะกว่า CoT เมื่อไม่มีคำตอบถูกผิดชัดเจน

4.5 ReAct Prompting

ReAct (Reasoning + Acting) คือเทคนิคที่ให้ AI สลับกันระหว่าง คิด (Think) และ กระทำ (Act) เช่น ค้นหาข้อมูล, เรียก API, รันโค้ด ใช้ใน AI Agent ที่ต้องทำงานหลายขั้นตอน

ReAct
Thought
"ต้องค้นหาข้อมูลราคาหุ้น NVDA วันนี้"
Action
search("NVDA stock price today")
Observation
ผล: "NVDA: $875.40 (+2.3%)"
Thought
"ได้ข้อมูลแล้ว ตอบผู้ใช้ได้"
Final Answer
"ราคาหุ้น NVIDIA วันนี้คือ $875.40 (+2.3%)"

ReAct ในทางปฏิบัติ

  • ใช้ใน AI Agent ที่เชื่อมต่อกับเครื่องมือ (Tools) เช่น Search, Calculator, Database
  • LangChain, LlamaIndex ใช้ ReAct เป็น Default Reasoning Strategy
  • ChatGPT Plugins และ Claude Tool Use ก็ใช้ ReAct-style
  • เหมาะกับงานที่ต้องการข้อมูล Real-time หรือทำงานหลายขั้นตอน

4.6 Self-Consistency Prompting

Self-Consistency คือการให้ AI สร้างคำตอบหลายรอบด้วยวิธีคิดที่แตกต่างกัน แล้วเลือกคำตอบที่ได้รับ Majority Vote สูงสุด ช่วยเพิ่มความน่าเชื่อถือสำหรับปัญหาที่มีคำตอบเดียว

Self-Consistency
คำถาม: 15% ของ 240 คือเท่าไร?
วิธีที่ 1
240 × 0.15 = 36
วิธีที่ 2
10% = 24, 5% = 12, รวม = 36
วิธีที่ 3
240 ÷ 100 × 15 = 36
Majority Vote: 36 (3/3 เห็นตรงกัน) → คำตอบสุดท้าย = 36
ผลการทดสอบ (Wang et al., 2022)
Self-Consistency เพิ่มความแม่นยำบน GSM8K Math benchmark จาก 56.5% (CoT alone) เป็น 74.4% — เพิ่มขึ้น ~18%

4.7 เลือกเทคนิคให้เหมาะกับงาน

เทคนิค เหมาะกับงาน ข้อดี ข้อเสีย Token ที่ใช้
Zero-shot งานทั่วไป, สรุป, แปล ง่าย ประหยัด Token ไม่แม่นในงานเฉพาะ ⭐
Few-shot จำแนกประเภท, รูปแบบเฉพาะ แม่นยำขึ้น กำหนด Format ได้ ต้องเตรียม Examples ⭐⭐
CoT คณิตศาสตร์, Logic, Reasoning แม่นยำสูงในงาน Reasoning ตอบช้า Token เพิ่ม ⭐⭐⭐
ToT กลยุทธ์, Creative, Complex สำรวจหลายทางเลือก ใช้ Token มาก ช้ากว่า ⭐⭐⭐⭐
ReAct Agent, Multi-step, Real-time ใช้ Tools ได้ ข้อมูล Real-time ต้องการ Tool Support ⭐⭐⭐⭐
Self-Consistency คำตอบเดียว, Math, Factual ความแม่นยำสูงสุด ใช้ Token มากที่สุด ⭐⭐⭐⭐⭐

4.8 สรุปและแบบฝึกหัด

สรุปสาระสำคัญ

  • Zero-shot — ถาม AI โดยตรง ไม่ให้ตัวอย่าง ง่ายและเร็ว
  • Few-shot — ให้ตัวอย่าง 2-5 ชุด ช่วยกำหนดรูปแบบและเพิ่มความแม่นยำ
  • CoT — "คิดทีละขั้น" ช่วยให้ AI แสดง Reasoning ก่อนตอบ แม่นยำในงาน Logic/Math
  • ToT — สำรวจหลายแนวทางพร้อมกัน เหมาะกับปัญหาซับซ้อน
  • ReAct — สลับ Thinking และ Acting ใช้ใน AI Agent ที่มี Tools
  • Self-Consistency — ตอบหลายรอบแล้ว Vote เลือก เพิ่มความแม่นยำสูงสุด

แบบฝึกหัดท้ายบท

1
เปรียบเทียบ Zero-shot vs Few-shot

เลือกงาน Classification (เช่น จำแนกประเภทข่าว) แล้วทดสอบทั้ง Zero-shot และ Few-shot (3 examples) วัดและเปรียบเทียบความแม่นยำ

2
CoT สำหรับโจทย์คณิตศาสตร์

นำโจทย์คณิตศาสตร์ 5 ข้อมาทดสอบ ทั้งแบบไม่ใช้ CoT และใช้ CoT ("คิดทีละขั้น") บันทึกและเปรียบเทียบผล

3
ToT สำหรับการตัดสินใจ

ใช้ ToT Prompt ช่วยตัดสินใจเรื่องที่คุณสนใจ (เช่น เลือกเรียนต่อ, เลือก Career Path) ให้ AI เสนอ 3 แนวทางพร้อมประเมินแต่ละทาง

4
Self-Consistency ทดลอง

ถามคำถาม Factual เดิม 5 ครั้งในหน้าต่างใหม่ทุกครั้ง บันทึกคำตอบ แล้วดูว่าคำตอบส่วนใหญ่ตรงกันไหม และมี Inconsistency ตรงไหนบ้าง
