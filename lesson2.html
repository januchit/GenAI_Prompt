2.1 Language Model คืออะไร?

Language Model (LM) คือโมเดลที่เรียนรู้ความน่าจะเป็นของคำหรือประโยคในภาษา กล่าวคือ โมเดลจะทำนายว่า "คำถัดไปคืออะไร" โดยอาศัยบริบทที่ผ่านมา

ตัวอย่างง่ายๆ
"ฉันชอบกิน ___" → โมเดลจะทำนายว่าคำที่เหมาะสมถัดไปคืออะไร เช่น "ข้าว", "ส้มตำ", "พิซซ่า" โดยแต่ละคำมีความน่าจะเป็นต่างกัน

วิวัฒนาการของ Language Model

ยุค 1990–2000s

N-gram Models

นับความถี่ของคำต่อเนื่องกัน n คำ ง่ายแต่ไม่เข้าใจบริบทระยะไกล

ยุค 2010s

RNN / LSTM

ใช้ Recurrent Neural Network จำบริบทได้ดีขึ้น แต่ช้าและลืมบริบทระยะไกล

2017–ปัจจุบัน

Transformer / LLM

Attention Mechanism ทำให้เข้าใจบริบทระยะไกลได้ดี เร็ว และ Scale ได้มหาศาล

2.2 Tokenization

ก่อนที่ LLM จะประมวลผลข้อความ ต้องแปลงข้อความเป็น Token ก่อน Token คือหน่วยย่อยของข้อความ ซึ่งอาจเป็นคำ, ส่วนของคำ, หรือสัญลักษณ์

ประโยค: "Generative AI เปลี่ยนโลก"

Gener ative AI เปล ี่ยน โลก

1 token ≈ 0.75 คำ ภาษาอังกฤษ  |  ภาษาไทย 1 token ≈ น้อยกว่า 1 พยางค์

วิธี Tokenization ยอดนิยม

Byte Pair Encoding (BPE)

ใช้ใน GPT-2, GPT-3, GPT-4, LLaMA รวมคู่ตัวอักษรที่ปรากฏบ่อยที่สุดเข้าด้วยกัน เหมาะกับหลายภาษา

WordPiece

ใช้ใน BERT, RoBERTa แบ่งคำออกเป็น subword ที่มีความน่าจะเป็นสูงสุด

SentencePiece

ใช้ใน T5, Gemma ทำงานระดับ byte ไม่ขึ้นกับช่องว่างระหว่างคำ เหมาะกับภาษาเอเชีย

Context Window
LLM รับ Token ได้จำกัดใน 1 การประมวลผล เรียกว่า Context Window เช่น GPT-4 = 128K tokens, Claude 3 = 200K tokens ยิ่ง Context Window กว้าง ยิ่งจำบริบทได้มากขึ้น

2.3 Word Embeddings

หลัง Tokenization แต่ละ Token จะถูกแปลงเป็น Embedding Vector — ตัวเลขหลายมิติที่แทนความหมายของคำ คำที่มีความหมายใกล้เคียงกันจะมี Vector ที่อยู่ใกล้กันในพื้นที่มิติสูง

แสดงว่า Embedding เข้าใจความสัมพันธ์เชิงความหมายได้จริง

Embedding Dimension

  • Word2Vec (2013) — 100–300 มิติ รุ่นแรกที่โด่งดัง
  • GloVe (2014) — 50–300 มิติ เรียนรู้จาก co-occurrence matrix
  • GPT-3 (2020) — 12,288 มิติ แต่ละ token เป็นเวกเตอร์ขนาดใหญ่มาก
  • Modern LLMs — หลายพัน dimension เพื่อจับความหมายที่ละเอียดขึ้น

Self-Attention คืออะไร?

กลไกหลักของ Transformer คือ Self-Attention ที่ทำให้โมเดลสามารถ "มองย้อนกลับ" ไปยังคำอื่นๆ ในประโยคได้พร้อมกัน

"The animal didn't cross the street because it was too tired."

Self-Attention ทำให้โมเดลรู้ว่า "it" หมายถึง "animal" ไม่ใช่ "street" โดยคำนวณ Attention Score ระหว่างทุกคำ

Positional Encoding

เนื่องจาก Transformer ประมวลผลคำทั้งหมดพร้อมกัน (ต่างจาก RNN ที่ประมวลผลทีละคำ) จึงต้องเพิ่ม Positional Encoding เพื่อบอกว่าคำแต่ละคำอยู่ตำแหน่งที่เท่าไร

  • Sinusoidal PE (ต้นฉบับ) — ใช้ฟังก์ชัน sine/cosine คงที่
  • Learned PE — เรียนรู้ตำแหน่งจากข้อมูล ใช้ใน GPT-2/3
  • RoPE (Rotary PE) — ใช้ใน LLaMA, Gemma รองรับ Context ยาวได้ดีกว่า

2.5 กระบวนการ Training LLM

การสร้าง LLM มี 3 ขั้นตอนหลัก:

1

Pre-training

ฝึกกับข้อมูลข้อความขนาดมหาศาล (Common Crawl, Wikipedia, GitHub, Books) ด้วยการทำนายคำถัดไป ใช้เวลาและพลังงานสูงมาก

Unsupervised Learning
2

Supervised Fine-tuning (SFT)

ปรับโมเดลด้วยข้อมูลที่มนุษย์เขียนตัวอย่าง ทั้งคำถามและคำตอบที่ดี เพื่อให้โมเดลตอบคำถามได้อย่างมีประโยชน์

Supervised Learning
3

RLHF (Reinforcement Learning from Human Feedback)

มนุษย์จัดอันดับคำตอบที่โมเดลสร้าง โมเดล Reward ถูกฝึก แล้วใช้ PPO เพื่อให้โมเดลหลักตอบได้ตรงความต้องการมนุษย์มากขึ้น

Reinforcement Learning

Scale Laws

Scaling Law (Kaplan et al., 2020)
ประสิทธิภาพของ LLM ขึ้นกับ 3 ปัจจัย: จำนวน Parameters (N), ขนาดข้อมูล (D), และ Compute (C) ยิ่งเพิ่มทั้ง 3 อย่างพร้อมกัน ประสิทธิภาพจะดีขึ้นตาม Power Law อย่างต่อเนื่อง

Emergent Abilities

เมื่อโมเดลใหญ่ถึงระดับหนึ่ง จะเกิดความสามารถที่ไม่ได้สอนโดยตรง เรียกว่า Emergent Abilities เช่น:

  • Chain-of-Thought Reasoning — คิดเป็นขั้นตอน
  • In-context Learning — เรียนรู้จากตัวอย่างใน Prompt โดยไม่ต้อง Fine-tune
  • Instruction Following — ทำตามคำสั่งที่ซับซ้อน
  • Code Generation — เขียนโปรแกรมได้โดยไม่ได้ฝึกเฉพาะด้าน

GPT vs BERT vs LLaMA

GPT (Decoder-only)

เหมาะกับงาน Generation สร้างข้อความ, ตอบคำถาม, เขียนโค้ด ใช้ Autoregressive — ทำนายคำทีละคำจากซ้ายไปขวา

BERT (Encoder-only)

เหมาะกับงาน Understanding จำแนกประโยค, NER, คำถาม-ตอบ ใช้ Masked LM — เห็นบริบทซ้าย-ขวาพร้อมกัน

T5 (Encoder-Decoder)

เหมาะกับงาน Seq2Seq แปลภาษา, สรุป, QA แปลง text เป็น text ได้ทุกรูปแบบ

2.7 สรุปและแบบฝึกหัด

สรุปสาระสำคัญ

  • Language Model ทำนายความน่าจะเป็นของคำถัดไป พัฒนาจาก N-gram → RNN → Transformer
  • Tokenization แปลงข้อความเป็น Token ก่อนป้อนให้โมเดล ภาษาไทยใช้ Token มากกว่าอังกฤษ
  • Embedding แปลง Token เป็น Vector เพื่อแทนความหมายในพื้นที่มิติสูง
  • Transformer ใช้ Self-Attention เข้าใจความสัมพันธ์ระหว่างคำในประโยคพร้อมกัน
  • Training LLM มี 3 ขั้น: Pre-training → SFT → RLHF
  • โมเดลสำคัญ: GPT-4o (OpenAI), Claude (Anthropic), Gemini (Google), LLaMA (Meta)

แบบฝึกหัดท้ายบท

1
ทดลอง Tokenizer

ลองใช้ platform.openai.com/tokenizer นับจำนวน Token ของประโยคภาษาไทยและอังกฤษ แล้วเปรียบเทียบผล

2
เปรียบเทียบโมเดล

ถามคำถามเดียวกันกับ ChatGPT, Claude และ Gemini แล้วเปรียบเทียบคุณภาพ ความถูกต้อง และสไตล์การตอบ

3
คำนวณ Context

ถ้าโมเดลมี Context Window 128K tokens และ 1 token ≈ 4 ตัวอักษร จะสามารถรับข้อความได้กี่หน้ากระดาษโดยประมาณ? (สมมติ 1 หน้า ≈ 3,000 ตัวอักษร)

4
ค้นคว้า

LLaMA 3 และ Mistral เป็น Open Source Model หาข้อมูลวิธีรัน LLM เหล่านี้บน computer ส่วนตัว โดยใช้ Ollama หรือ LM Studio
